正确地处理换行

在 Markdown 源代码中，段落由两个或多个空行分隔，Typora 会保证每次按下 Enter 键时即可创建新的段落（即含有两个换行），如果只需要一个换行，使用 Shift+Enter 键一次，之后的 Enter 的默认行为都会只有一个换行，持续到编辑器退出为止。

为了保证与其他 Markdown 解析器兼容（正确地识别换行符），建议：

  1. 保留两个换行（即 Typora 的默认换行行为）
  2. 行尾保留两个空格，再换行一次
  3. 插入 <br />

正确地处理图表

Typora 支持图表的某些 Markdown 扩展。当导出为 HTML，PDF，epub，docx 时，这些渲染图也将包括在内，但将 Markdown 导出为当前版本的其他文件格式时，不支持图功能。

此外，标准的 Markdown 或 CommonMark 或 GFM 不支持图表。因此，仍然建议插入这些图的图像，而不是直接将它们写在 Markdown 中。
